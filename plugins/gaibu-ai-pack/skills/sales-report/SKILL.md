# /sales-report — 売上集計・分析スキル

## トリガー

`/sales-report` と入力、または「売上を分析して」「CSVを分析して」と言われた時に起動する。

## 前提条件

- POSシステム・売上管理ソフトからエクスポートしたCSVファイルが必要
- CSVをチャットに添付してもらう
- 添付がない場合は「売上CSVを添付してください」と案内する

## 実行手順

### Step 1: CSVの確認
添付されたCSVを開き、以下を確認する：
- 日付列の形式
- 売上金額の列名
- その他の列（客数・スタッフ・商品名等）

### Step 2: 集計
以下を計算する：
```
- 期間合計売上
- 1日平均売上
- 曜日別売上（月〜日）
- スタッフ別売上（列があれば）
- 商品・メニュー別売上ランキング（列があれば）
- 客数・客単価（列があれば）
```

### Step 3: 比較・インサイト
```
- 前月比（前月データがあれば）
- 売上が多い日・少ない日の傾向
- 売れ筋TOP5・不振商品
- 曜日・時間帯のパターン
```

### Step 4: レポート出力
以下のフォーマットでMarkdown出力する：

```markdown
# 売上レポート｜〇〇年〇月

## ▼ サマリー（1分で読める）
| 指標 | 今月 | 前月 | 増減 |
|---|---|---|---|
| 売上合計 | 〇,〇〇〇,〇〇〇円 | | |
| 1日平均 | 〇〇〇,〇〇〇円 | | |
| 客数合計 | 〇〇〇人 | | |
| 客単価 | 〇〇,〇〇〇円 | | |

## ▼ 曜日別売上
（表形式で出力）

## ▼ 売れ筋TOP5
1. 〇〇 — 〇〇〇,〇〇〇円（構成比 〇%）
...

## ▼ 注目すべき傾向
- 〇〇曜日の売上が突出して高い（要因を分析）
- 〇〇の売上が前月比-〇%で低下

## ▼ 推奨アクション
1. 〇〇曜日のキャンペーン実施を検討
2. 〇〇スタッフの〇〇強化
3. 〇〇メニューの見直し
```

## 制約

- POSシステムへの直接接続は不可（CSV添付が前提）
- グラフはテキスト形式のみ（画像グラフ不可）
- 前月比は前月データがある場合のみ計算
