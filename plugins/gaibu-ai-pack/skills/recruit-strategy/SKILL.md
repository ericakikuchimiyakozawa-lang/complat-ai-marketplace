# /recruit-strategy — 採用戦略・採用ファネル・面接設計スキル

## トリガー

`/recruit-strategy` と入力、または「採用がうまくいかない」「応募が来ない」「面接で何を聞けばいいか」「採用の全体設計をしたい」「採った人がすぐ辞める」と言われた時に起動する。`complat-hiring` エージェントが担当する（求人原稿だけの場合は `/job-posting`）。

## 前提条件

- データ不要。ヒアリングで進める
- 最低限ヒアリングするもの：業種／募集職種／現在の採用課題（応募が来ない／採れても辞める 等）／求める人物像／予算

## 実行手順

### Step 1: 採用ファネルの現状把握
「認知 → 応募 → 面接 → 内定 → 入社 → 定着」のどこで詰まっているかを特定する。

### Step 2: ボトルネック別の打ち手
| 詰まり | 主な原因 | 打ち手 |
|---|---|---|
| 応募が来ない | 媒体・原稿・条件 | 媒体選定・求人原稿改善（→ /job-posting）・条件見直し |
| 面接で辞退 | 連絡遅い・魅力不足 | スピード対応・職場の魅力の言語化 |
| 採っても辞める | ミスマッチ・受け入れ | 面接の見極め・オンボーディング設計 |

### Step 3: 面接設計
求める人物像（行動指針との一致）を見極める質問リストを設計する。スキルだけでなく価値観・定着可能性を見る質問を入れる。

### Step 4: オンボーディング・定着設計
入社後30日／90日の受け入れ計画、教育担当、初期の不安解消の仕組みを設計する。

### Step 5: アウトプット
採用の全体設計（ファネル＋打ち手＋面接質問＋オンボーディング）を1枚にまとめる。

## 出力フォーマット

```markdown
# 採用戦略｜〇〇（職種）

## ▼ 採用ファネルの現状とボトルネック
## ▼ ボトルネック別の打ち手
## ▼ 面接質問リスト（見極めポイント付き）
## ▼ オンボーディング（30日／90日）
## ▼ 今月のアクション3つ
```

## 制約

- 「求人原稿を直す」だけで終わらせず、ファネル全体（定着まで）で設計する
- 面接質問は必ず「何を見極めるための質問か」をセットで示す
- 求人原稿そのものの生成は `/job-posting` に引き継ぐ
- 採用基準と評価制度の一貫性が必要なら `complat-hr-strategy` に連携する
