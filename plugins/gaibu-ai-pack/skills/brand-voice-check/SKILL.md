# ブランドボイス点検

クライアントの文章成果物が、その案件の `brand-voice.md` に沿っているかを点検する。

## 手順
1. 案件フォルダ直下（またはカレント階層を上に辿って）`brand-voice.md` を読む。見つからなければ、どこにあるかをユーザーに確認する。
2. 対象テキストを以下の観点で照合する：
   - 一人称・口調・敬体/常体
   - 禁止表現・NGワード
   - 文章構造ルール（一文の長さ、リズム、見出しの作法など）
   - トーン（煽らない／事実ベース 等、案件の指定に従う）
3. 逸脱箇所を「該当箇所 → 問題 → 修正案」の3点セットで列挙する。
4. 求められれば、修正適用後の全文を提示する（既存ファイルの上書きは承認後）。

## 出力フォーマット
- まず逸脱の有無を一言で結論。
- 次に逸脱リスト（なければ「逸脱なし」）。
- 最後に必要なら修正版。

brand-voice.md が存在しない案件では、勝手に基準を作らず、基準が未整備である旨を明示する。
